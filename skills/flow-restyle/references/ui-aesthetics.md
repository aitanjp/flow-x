# UI 美学决策框架（精简版）

> 融合自 Anthropic 原版 `frontend-design` skill + 社区 `impeccable` 项目精华。
> **本文件是 flow-x 内置的"够用"基线**。

## 决策的 4 个问题（开工前必须想清楚）

### 1. 目的（Purpose）
- 这个界面解决什么问题？
- 用户在什么场景下打开它？
- 用户期待的核心动作是什么（看 / 选 / 填 / 决策）？

### 2. 调性（Tone）

**选一个明确的调性方向**，不允许"现代风"这种空话。下面是 9 种主流方向。

> **关键**：粗野和精致都能赢，关键是"明确选一个并执行到位"。

---

#### 1️⃣ 编辑式（Editorial）
- **关键词**：杂志感 / 衬线显示字 / 大留白 / 长文阅读 / 暖纸色
- **参考**：A List Apart、Apple Newsroom、Pitchfork
- **适合**：内容驱动 / 品牌权威 / 长文 / 文化产品
- **不适合**：高频操作工具 / dashboard

#### 2️⃣ 极简（Minimal）
- **关键词**：单色 / 几何字 / 无装饰 / 留白 / 净中性
- **参考**：Linear、Vercel、Stripe、Notion 静态页
- **适合**：技术工具 / B2B SaaS / 开发者产品
- **不适合**：需要情感连接的消费场景

#### 3️⃣ 工业（Industrial）
- **关键词**：等宽字 / 暴露网格 / 单调 / 冷色 / 数据密度
- **参考**：Bloomberg Terminal、Grafana、Datadog
- **适合**：监控 / 金融 / 工程师工具
- **不适合**：面向大众 / 情感场景

#### 4️⃣ 玩具（Playful）
- **关键词**：圆角 / 高饱和 / 弹动 / 插画 / 渐变可控
- **参考**：Duolingo、Headspace（活力页）
- **适合**：教育 / 儿童 / 协作 / 习惯养成
- **不适合**：严肃 / 专业 / 金融

#### 5️⃣ 奢华（Luxury）
- **关键词**：衬线 / 金属质感 / 深色 + 金 / 大留白 / 慢节奏
- **参考**：Bottega Veneta、Aesop、Apple Vision Pro 落地页
- **适合**：高端品牌 / 奢侈品 / 私人服务
- **不适合**：大众消费 / 工具

#### 6️⃣ 有机（Organic）
- **关键词**：流动形 / 暖色 / 不规则 / 自然意象 / 柔和过渡
- **参考**：Headspace、Calm、Apple Health、Oatly
- **适合**：健康 / 冥想 / 医疗-关怀 / 食品 / 母婴
- **不适合**：商业仪表盘 / 数据密集

#### 7️⃣ 复古未来（Retro-Futurism）
- **关键词**：80s/90s / 像素 / 霓虹 / 方块 / CRT 噪点
- **参考**：Cyberpunk 2077 落地页、Vercel Conf
- **适合**：游戏 / 创意工具 / Web3
- **不适合**：严肃业务 / 老年用户

#### 8️⃣ 粗野（Brutalist）
- **关键词**：单一字 / 黑白 / 网格暴露 / 反美学 / 大块色
- **参考**：Balenciaga 早期、独立设计师作品集
- **适合**：设计圈 / 反主流 / 艺术 / 文化机构
- **不适合**：大众用户 / 转化漏斗

#### 9️⃣ 极繁（Maximalist）
- **关键词**：信息密集 / 多层叠加 / 装饰主义 / 多字体
- **参考**：Eye Magazine、It's Nice That（活动页）
- **适合**：内容广场 / 节日主题 / 文化展览
- **不适合**：日常工具 / 长期阅读

**推荐生成规则**：
- 必须给 1 首选 + 1 备选
- 推荐理由必须**结合项目的具体业务**
- 必须显式排除明显不合适的
- 用户已有强偏好 → 直接锁定对应方向

### 3. 约束（Constraints）
- 技术栈：什么框架？SSR / SPA？
- 性能：首屏 LCP 目标？
- 无障碍：WCAG 等级要求？
- 品牌限定：已有 logo / 色板 / 字体？哪些不可改？

### 4. 差异化（Differentiation）
- 这个界面让人记住的"那一件事"是什么？
- 三天后用户描述这个产品时会用到的形容词？
- 如果只能保留一个视觉决策，是哪个？

## 五大美学维度（实施层）

### 1. 字体（Typography）
- **选一对**：display 字体 + body 字体
- **绝对避开** AI slop 默认：Inter / Roboto / Arial / system-ui（除非品牌强制）
- **行高**：body **1.6**
- **行长**：body 限制 65–75ch
- **流体头部**：headings 用 `clamp()`

### 2. 颜色（Color）
- **用 OKLCH**，不用 hex
- **倾斜中性色**：纯灰会"机器感"。给中性色一点点品牌色 hue
- **一个声音原则**：主色调只有 1 个，第二个强调点**用粗细或字号**
- **暗色不等于黑**：避免 #000 / #fff

### 3. 动效（Motion）
- **缓动曲线**：expo-out（`cubic-bezier(0.16, 1, 0.3, 1)`）
- **禁止 bounce / elastic**
- **时长档位**：颜色 150ms；transform 300–400ms；编排入场 600–1200ms
- **只动 transform 和 opacity**
- **prefers-reduced-motion**：必须支持

### 4. 空间布局（Spatial）
- **不对称比对称好**
- **间距比例**：8/16/24/32/48/80/120 非线性 scale
- **密度二选一**：要么大留白，要么受控密度，不要中间态

### 5. 背景与质感（Atmosphere）
- 别只用纯色背景：渐变网格、噪点纹理、几何图案、分层透明度、戏剧性阴影

## 阴影词汇

- **平静默认**：组件 at rest 不带阴影
- **状态响应**：阴影只在 hover / focus / 主动抬起时出现
- **低 alpha**：每个阴影最强 blur 的 alpha ≤ 0.15
